# Search Patterns

ユーザーの職種・ロールに応じた検索クエリ最適化の参照表。
Step 2 でユーザーの職種が判明したとき、該当カテゴリのクエリを選んで使う。
`{}` は実際のテーマで置換する。日本市場向けの内容は日本語クエリも併用すること。

複数職種にまたがる場合（例: PM × エンジニア）は両カテゴリのクエリを組み合わせる。

## ヒアリング指針

職種・ロールが不明な場合は Step 1 のテーマ確認時に以下を尋ねる。

- 「どんなお仕事をされていますか？（職種・ロール）」
- 「このスライドは誰に見せますか？（社内向け／顧客向け／勉強会など）」

## ロール推定ヒント（会話履歴からの職種シグナル）

Step 1-B でユーザーの職種が不明な場合、会話履歴中のキーワードを以下の対応表で照合し、
ロールを推定する。複数シグナルが混在する場合は最も多く出現したカテゴリを採用する。

| シグナルワード（例） | 推定ロール |
|--------------------|-----------|
| デプロイ、CI/CD、プルリク、リファクタ、テスト、マージ、ブランチ、コンテナ、Kubernetes、Docker | ソフトウェアエンジニア |
| アーキテクチャ、技術選定、ADR、スプリント、ベロシティ、チームビルディング | エンジニアリングマネージャー・テックリード |
| 特徴量、モデル精度、過学習、混同行列、推論、データパイプライン | データサイエンティスト |
| Figma、ワイヤーフレーム、ユーザビリティ、コンポーネント、デザインシステム | デザイナー |
| CVR、LTV、CTR、ファネル、アトリビューション、広告、CPA | マーケティング |
| バックログ、ロードマップ、RICE、KPI、ステークホルダー、ユーザーストーリー | プロダクトマネージャー |
| ARR、MRR、チャーン、バーンレート、CAC、ピッチ、エクイティ | 創業者・経営者 |
| 案件、商談、クロージング、提案書、トークスクリプト、受注 | 営業 |
| 予算、P&L、減価償却、キャッシュフロー、取締役会 | 財務 |
| フレームワーク、課題分析、論点整理、MECE、イシューツリー | コンサルタント |
| 採用、オンボーディング、評価制度、組織設計、1on1 | 人事 |
| 契約書、NDA、規約、コンプライアンス、法的リスク | 法務 |
| 論文、実験、仮説、エビデンス、サンプル数 | 研究者・科学者 |
| カルテ、患者、診断、治療方針、インフォームドコンセント | 医療・ヘルスケア |
| 授業、教材、生徒、板書、シラバス | 教育者 |
| レポート、課題、ゼミ、参考文献、論述 | 学生 |
| 原稿、執筆、取材、編集、ライティング | ライター |

### 推定精度向上のポイント

- シグナルが2個以上ヒットしたカテゴリを優先する
- 「エンジニア」「PM」などの職種名が明示されている場合はそちらを優先する
- シグナルが1つも該当しない場合は「汎用（クロス職種）」として扱い、
  ヒアリング指針に従いユーザーに確認する
