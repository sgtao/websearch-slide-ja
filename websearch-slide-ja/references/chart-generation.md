# Chart Generation Rules

Web 検索や会話から得た数値データを SVG グラフ化する際のルール・判断基準。
**Step 3-0（グラフ化判定）**と **Step 4（HTML 生成）** で参照すること。

---

## グラフ化判断フロー

以下 4 条件を順に判定し、**すべて YES** の場合のみグラフを生成する。
1 つでも NO があれば、テキスト・表で代替する。

1. **数値性** — テーマに数値比較・推移・構成比のいずれかが主役として含まれるか？
   - YES 例: 「人気拡張機能 ダウンロード数」「クラウドシェア」「2020-2025 のスター推移」
   - NO 例: 「Claude Code の使い方 Tips」「React ベストプラクティス」（数値が主役でない）
2. **データ点数** — 信頼できる数値が **3 点以上** 取得できたか？
   - 3 点未満 → グラフ不要（テキスト記述で十分）
3. **単位と出典の明確性** — すべての数値に **単位・取得時点・出典** がセットで揃うか？
   - 1 つでも欠ければグラフ化しない（推測値の混入は厳禁）
4. **グラフ種別の判定可能性** — 下表「データ形状 → グラフ種別マッピング」で 1 種類に絞れるか？
   - 絞れない場合は表形式に切り替え

> **モード制約**: 「会話のみモード（3 枚ミニ含む）」では Web 検索の数値・出典が得られないため、
> 原則として **グラフ生成は実施しない**。会話内に明示的な数値・単位・出典がすべて揃っている場合のみ例外的に許可する。

---

## データ形状 → グラフ種別マッピング

| データ形状 | データ例 | 推奨テンプレート | 系列数の目安 |
|---|---|---|---|
| カテゴリ × 数値（順序付き） | 拡張機能トップ 5 の DL 数 | `bar-chart.svg` | 1 系列・3〜10 件 |
| 時系列の数値変化 | 2020-2025 の GitHub スター推移 | `line-chart.svg` | 1〜3 系列・3〜10 点 |
| 構成比（合計 100%） | クラウドベンダー市場シェア | `donut-chart.svg` | 2〜5 系列 |
| 数値ペアの相関 | スター数 × コミット数 | `scatter-plot.svg` | 1〜2 系列・5〜20 点 |
| 上記に該当しない / 多次元 | 拡張機能ごとの複数評価軸 | 表形式に切り替え | — |

## データ抽出ルール

Web 検索結果から数値を取り出す際、以下の形式で記録してから SVG に埋め込む。
取り出しと描画を分けることで、出典抜け・単位ミスマッチを事前に検出できる。

```
[グラフデータ]
種別:       bar-chart
タイトル:   人気VSCode拡張機能 ダウンロード数（2025年5月時点）
データ:
  - Python:        95000000
  - Live Server:   41000000
  - Prettier:      38000000
  - ESLint:        32000000
  - GitLens:       27000000
単位:       ダウンロード回数
出典:       Visual Studio Marketplace
出典URL:    https://marketplace.visualstudio.com/
取得時点:   2025-05
```

> 数値抽出の段階で **1 つでも単位・出典が欠ける** ことに気づいたら、その時点でグラフ化を諦める。

## 出典明記ルール（必須）

すべてのグラフ SVG に以下の要素を必ず含める。省略は禁止。

| 要素 | SVG 表現 | 省略可否 |
|---|---|---|
| グラフタイトル | `<text class="chart-title">{タイトル}</text>` | 必須 |
| X 軸ラベル | `<text class="chart-axis-label">` | 必須（軸を持つチャート） |
| Y 軸ラベル | `<text class="chart-axis-label" transform="rotate(-90 ...)">` | 必須（軸を持つチャート） |
| 数値ラベル | 各データ点付近に `<text class="chart-value">` | 必須 |
| 凡例 | 2 系列以上の場合 `<text class="chart-legend">` | 2 系列以上で必須 |
| 出典 | SVG 右下に `<text class="chart-source">出典: {サイト名}</text>` | 必須 |

## HTML への埋め込み方法

SVG は **インライン展開** する（`<img src="*.svg">` は不可）。
CSS 変数 `var(--chart-N)` を効かせるには SVG が DOM 内に直接存在する必要がある。

**禁止**: Chart.js・D3.js・ECharts 等の外部グラフライブラリは一切使用しない。
- `<canvas>` ベースのグラフも禁止。グラフは必ず SVG で記述する。
- 外部 CDN スクリプトを追加することも禁止（スキルの自己完結性を維持するため）。

```html
<section class="slide">
  <h2>人気VSCode拡張機能 トップ5（DL数）</h2>
  <div class="slide-chart">
    <svg viewBox="0 0 720 360" xmlns="http://www.w3.org/2000/svg" class="chart-svg">
      <!-- assets/chart-templates/bar-chart.svg の内容をコピーして {} 部を埋める -->
    </svg>
  </div>
</section>
```

### 座標計算ヒント

各テンプレートの ViewBox は `0 0 720 360`。本文領域 ≒ `160-680, 50-310`。

| グラフ | X 計算 | Y 計算 |
|---|---|---|
| bar（横棒） | `width = (val / max) × 480` | カテゴリ行で固定間隔 |
| line | `x = 160 + (i / (N-1)) × 480` | `y = 290 - ((val - min) / (max - min)) × 220` |
| donut | 中心 (260, 180)、外径 110、内径 60 | 12 時方向起点で時計回り |
| scatter | `x = 160 + ((vx - xmin) / (xmax - xmin)) × 480` | `y = 290 - ((vy - ymin) / (ymax - ymin)) × 220` |

## 制約事項

- 1 スライドに含めるグラフは **最大 1 つ**
- 同一スライドに画像（`<figure class="slide-figure">`）とグラフ（`<div class="slide-chart">`）を共存させない（密度過多）
- 系列は最大 5 まで（`--chart-1` 〜 `--chart-5`）。6 以上は表に切り替え
- グラフ内テキストは `font-family: inherit` で本文フォント（Noto Sans JP）を継承
- 色のハードコード禁止 — 必ず `chart-series-N` / `chart-line-N` クラス経由

## 生成後チェックリスト

- [ ] グラフタイトルが明記されているか
- [ ] 軸ラベル（X 軸・Y 軸）が明記されているか（軸を持つチャート）
- [ ] 全データ点に数値ラベルが付いているか
- [ ] 凡例（2 系列以上の場合）があるか
- [ ] 出典が SVG 右下に明記されているか
- [ ] 色指定にハードコードがないか（`fill="#..."` がなく、すべてクラス経由）
- [ ] ライト／ダーク両モードで読みやすいか（コントラスト確認）
- [ ] 同一スライドに画像とグラフが共存していないか
- [ ] 1 スライド = 最大 1 グラフを守れているか
