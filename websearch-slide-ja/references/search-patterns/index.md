# Search Patterns — インデックス

クロス職種（汎用）テーマのクエリと、ロール別ファイルへのマップ。

`{}` は実際のテーマで置換する。日本市場向けは日本語クエリも併用すること。
複数職種にまたがる場合（例: PM × エンジニア）は両カテゴリのファイルを読む。

---

## ロール別ファイルマップ

| ロールカテゴリ | 参照ファイル | 含まれるロール |
|--------------|------------|--------------|
| 技術系 | `search-patterns/tech.md` | エンジニア / EM・テックリード / データサイエンス / デザイナー |
| ビジネス系 | `search-patterns/business.md` | PM / 経営 / マーケ / 営業 / 財務 / コンサル / オペレーション / 人事 |
| 専門職 | `search-patterns/specialist.md` | 法務 / 研究者 / 医療 / 教育者 / 学生 / ライター |

---

## クロス職種（汎用テーマ）

職種が特定できない場合、またはテーマが職種横断の場合に使う。

| テーマ | 主な検索クエリ |
|--------|--------------|
| Claude Code / AI開発 | `claude code skill github stars`, `claude code tips x.com` |
| MCP サーバー | `MCP server awesome list github`, `model context protocol trending` |
| Python / JS ライブラリ | `{library} examples github`, `{library} tricks 2025` |
| AI ツール比較 | `{tool} vs {tool} github`, `{tool} popular use cases` |
| 業界トレンド | `{industry} trends 2025`, `state of {field} report` |
| ベストプラクティス | `{topic} best practices 2025`, `{topic} common pitfalls` |
